## Apache Kafka — Queue + Pub/Sub 통합

Kafka는 **로그 기반 분산 이벤트 스트리밍 플랫폼**으로, Queue와 Pub/Sub 두 패턴을 하나의 모델로 통합합니다.

<KeyTable
  columns={["개념", "설명"]}
  rows={[
    [
      "Topic · Partition",
      "메시지가 토픽에 append-only로 저장된다. 토픽은 여러 파티션으로 나뉘어 병렬 처리가 가능하다.",
    ],
    [
      "Consumer Group",
      "같은 그룹 내 컨슈머끼리는 파티션을 나눠 가져 Queue처럼 동작한다 (1메시지 → 1컨슈머).",
    ],
    [
      "여러 Consumer Group",
      "서로 다른 그룹은 같은 메시지를 각각 독립적으로 읽어 Pub/Sub처럼 동작한다.",
    ],
    [
      "메시지 보존",

    ],
    [
      "처리량",

    ],
  ]}
/>

## MQTT — IoT 특화 Pub/Sub 프로토콜

MQTT(Message Queuing Telemetry Transport)는 **저대역폭·불안정한 네트워크 환경**에서도 동작하도록 설계된 경량 Pub/Sub 프로토콜이다. IoT 디바이스, 센서, 모바일 앱 등 리소스가 제한된 환경에서 널리 사용된다.

<KeyTable
  columns={["개념", "설명"]}
  rows={[

    ],
    [
      "Topic",
      "계층형 문자열 (예: home/living-room/temperature). 와일드카드(+, #)로 여러 토픽을 한 번에 구독할 수 있다.",
    ],
    [
