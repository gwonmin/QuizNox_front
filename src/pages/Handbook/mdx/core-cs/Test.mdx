## CI에서의 테스트

- **실패 시 동작:** 테스트가 하나라도 실패하면 보통 파이프라인을 실패로 처리하고, 배포 단계로 넘기지 않는다.
- **DevSecOps:** 테스트 다음에 보안 스캔(의존성·이미지·시크릿 검출 등)을 두고, 둘 다 통과한 경우에만 아티팩트를 푸시·배포하도록 하면 취약한 버전이 나가는 것을 막을 수 있다.
- **Flaky 테스트:** 가끔만 실패하는 테스트는 신뢰를 떨어뜨리므로, 원인 제거(타이밍·순서·공유 상태) 또는 일시 스킵 후 수정하는 것이 좋다.
- **병렬화:** 테스트 수가 많으면 스위트를 나누어 병렬 실행해 전체 소요 시간을 줄인다.
