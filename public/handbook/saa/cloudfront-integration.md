# CloudFront 연동

**CDN**으로 캐시·배포를 앞단에 두는 패턴입니다. 정적·동적 콘텐츠 캐싱, 오리진(S3, ALB 등)과 연동해 지연·트래픽을 줄입니다.

---

## 특징

- **엣지 캐시**: 전 세계 엣지에서 정적·동적 콘텐츠 캐시, 오리진(S3, ALB, 커스텀)에서 캐시 미스 시 조회
- **연동**: S3 정적 호스팅 앞단, ALB 오리진으로 동적 요청 라우팅
- **보안**: HTTPS·도메인·인증서 연결, OAC(Origin Access Control)로 S3 직접 접근 차단 가능

---

## 시나리오

| 조건 | 선택 |
|------|------|
| 정적 웹·글로벌 지연 감소 | S3 + CloudFront |
| **정적 + 동적 둘 다·단일 도메인** | **CloudFront 배포 하나**에 **S3(정적)·ALB(동적)** 를 오리진으로 등록 → Route 53에서 해당 배포로 라우팅 |
| 동적 요청·API | ALB를 오리진으로 CloudFront 구성 |
| 웹 공격 방어 | CloudFront + WAF |
| **전역 지연 감소·비용 최소**(정적 사이트) | S3 + CloudFront; Transfer Acceleration·Global Accelerator보다 일반적으로 비용·구성 단순 |

---

## 기출빈출

| 시나리오 | 접근 |
|----------|------|
| 정적+동적 단일 도메인 | **CloudFront**에 S3(정적)·ALB(동적) 오리진 등록. |
| 전역 지연·정적 사이트 | **S3 + CloudFront**; OAC로 S3 직접 접근 차단. |
| 웹 공격 방어 | **CloudFront + WAF**. |
