# EC2 vs Lambda vs ECS vs EKS

**Compute 선택** 시 비교하는 네 가지 옵션입니다. VM 직접 관리부터 서버리스·컨테이너까지 요구사항에 맞게 선택합니다.

---

## 특징

| 구분 | EC2 | Lambda | ECS / EKS |
|------|-----|--------|-----------|
| **단위** | 가상 서버(VM) | 함수(이벤트 기반) | 컨테이너 오케스트레이션 |
| **관리** | OS·패치·확장 직접 제어 | 런타임·확장 AWS 책임 | 컨테이너·클러스터 관리 |
| **운영 부담** | 높음 | 낮음 | 중간 |
| **적합 워크로드** | 유연한 범용·상시 구동 | 짧은 요청·이벤트·간헐적 | 마이크로서비스·컨테이너 상시 |

---

## 시나리오

| 조건 | 선택 |
|------|------|
| 최대 유연성·기존 VM 기반 | EC2 |
| 이벤트 기반·짧은 실행·최소 운영 | Lambda |
| 컨테이너·마이크로서비스·Kubernetes 표준 | ECS 또는 EKS |
| "최소 운영" 강조 | Lambda·Fargate·관리형 서비스 |

---

## 기출빈출

| 시나리오 | 접근 |
|----------|------|
| 최소 운영·이벤트 기반 | **Lambda**·Fargate. |
| 컨테이너·Kubernetes 표준 | **EKS**; AWS 네이티브면 **ECS**. |
| 유연성·VM 직접 제어 | **EC2** + ASG. |

---

## 요약

| 항목 | 설명 |
|------|------|
| EC2 | VM, 유연성 높음·운영 부담 큼 |
| Lambda | 함수·서버리스, 자동 확장·사용량 과금 |
| ECS/EKS | 컨테이너, ECS=AWS 네이티브·EKS=Kubernetes |
| 조건 | "서버리스" → Lambda / "컨테이너" → ECS·EKS |
