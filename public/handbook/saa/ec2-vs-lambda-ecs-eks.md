# EC2 vs Lambda vs ECS vs EKS

**Compute 선택** 시 비교하는 네 가지 옵션입니다.

## EC2

- **가상 서버** 직접 관리. OS·패치·확장 직접 제어.
- 유연성 최대, 운영 부담 큼.

## Lambda

- **이벤트 기반**, 서버 없이 함수만 실행. 자동 확장, 사용한 만큼 과금.
- 짧은 요청·이벤트 처리에 적합.

## ECS / EKS

- **컨테이너** 오케스트레이션. ECS는 AWS 네이티브, EKS는 Kubernetes.
- 마이크로서비스·컨테이너 워크로드에 사용.

## 요약

| 구분 | EC2 | Lambda | ECS/EKS |
|------|-----|--------|---------|
| 단위 | VM | 함수 | 컨테이너 |
| 운영 부담 | 높음 | 낮음 | 중간 |
