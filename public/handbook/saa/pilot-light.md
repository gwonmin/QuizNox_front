# Pilot Light (DR)

**핵심 구성만** 대기 리전에 상시 준비해 두는 DR 패턴입니다. DB 복제·핵심 데이터만 대기 리전에 두고, 장애 시 나머지 리소스(EC2 등)를 기동해 전환합니다.

---

## 특징

- **구성**: DB 복제·핵심 데이터만 대기 리전에 유지, 컴퓨트는 최소 또는 미기동
- **전환**: 장애 시 **나머지 리소스(EC2 등)** 기동 후 트래픽 전환
- **비용**: Active-Active보다 낮고, Backup보다 RTO 짧음

---

## 시나리오

| 조건 | 선택 |
|------|------|
| RTO를 Backup보다 줄이고 비용은 낮게 | Pilot Light |
| RTO 더 줄이고 대기 환경 상시 가동 | Warm Standby |
| 최소 RTO·RPO | Active-Active |

---

## 기출빈출

| 시나리오 | 접근 |
|----------|------|
| RTO 단축 + 비용 절감 | **Pilot Light**: 핵심(DB 등)만 대기, 장애 시 나머지 기동 후 전환. |

---

## 요약

| 항목 | 설명 |
|------|------|
| Pilot Light | 핵심만 대기, 장애 시 나머지 기동 후 전환 |
| RTO | Backup보다 짧음 |
| 조건 | "RTO 단축 + 비용 절감" → Pilot Light |
