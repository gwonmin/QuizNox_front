# Warm Standby (DR)

**축소된 대기 환경**을 상시 켜 두는 DR 패턴입니다. 대기 리전에 작은 규모 인스턴스·리소스를 항상 실행하고, 장애 시 스케일 업·트래픽 전환으로 복구합니다.

---

## 특징

- **구성**: 대기 리전에 **작은 규모** 인스턴스·리소스를 항상 실행
- **전환**: 장애 시 **스케일 업·트래픽 전환**으로 복구
- **RTO**: Pilot Light보다 짧고, Active-Active보다 비용·복잡도 낮음

---

## 시나리오

| 조건 | 선택 |
|------|------|
| RTO를 Pilot Light보다 줄이고 비용은 제한 | Warm Standby |
| 최소 RTO·RPO·글로벌 분산 | Active-Active |
| RTO 길어도 됨·비용 최소 | Backup·Pilot Light |

---

## 기출빈출

| 시나리오 | 접근 |
|----------|------|
| RTO 단축 + 상시 대기(축소 규모) | **Warm Standby**: 대기 리전에 작은 규모 상시 실행, 전환 시 스케일 업. |

---

## 요약

| 항목 | 설명 |
|------|------|
| Warm Standby | 축소된 대기 환경 상시 가동, 전환 시 스케일 업 |
| RTO | Pilot Light보다 짧음 |
| 조건 | "RTO 단축 + 상시 대기" → Warm Standby |
