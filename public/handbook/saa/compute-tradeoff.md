# Compute 선택: 운영 부담 vs 확장성 vs 비용

Compute 선택 시 **트레이드오프**를 이해하는 프레임입니다. "최소 운영", "비용 최적화", "확장성" 중 무엇을 우선할지에 따라 옵션이 달라집니다.

---

## 특징

- **운영 부담**: EC2 > ECS/EKS > Lambda 순으로 관리할 것이 많음. Lambda는 런타임·패치 대부분 AWS 책임.
- **확장성**: Lambda·ECS/EKS는 자동 확장 용이. EC2는 ASG로 확장. 짧은 트래픽 폭은 Lambda, 지속·컨테이너 워크로드는 ECS/EKS.
- **비용**: 항상 켜진 워크로드는 EC2·예약이 유리할 수 있음. 간헐적·피크 트래픽은 Lambda·온디맨드가 유리할 수 있음.

---

## 시나리오

| 조건 | 선택 |
|------|------|
| 운영 줄이기·관리 최소화 | Lambda·Fargate·관리형 서비스 |
| 안정 워크로드·비용 절감 | EC2 Reserved Instance·Savings Plans |
| 트래픽 변동 큼·피크 대응 | Lambda·ASG·온디맨드 |
| 중단 가능·배치 작업 | Spot |
