# SG / NACL 분석 (트러블슈팅)

**트래픽 차단** 원인을 **Security Group(SG)**과 **Network ACL(NACL)** 기준으로 짚는 방법입니다.

---

## 특징

- **NACL**: 서브넷 경계, **Stateless**. **인바운드·아웃바운드 둘 다** 허용 규칙 필요. **Ephemeral 포트**(1024–65535) 아웃바운드 허용이 없으면 응답 트래픽 차단.
- **Security Group**: 인스턴스/ENI, **Stateful**. 인바운드만 허용하면 응답 아웃바운드는 자동. **Private 서브넷**에서 인터넷 나가려면 **아웃바운드** 80/443 등 허용.
- **순서**: NACL(서브넷) → SG(인스턴스). **Flow Logs**로 accept/reject 확인 후 NACL·SG 규칙 대조.
- **새 서브넷** 추가 시: DB 서브넷 NACL에 **새 웹 서브넷 CIDR**에 대한 인바운드(3306)·아웃바운드(ephemeral) 필요.
- **퍼블릭 웹** 접속 불가: SG 인바운드 80/443, NACL 인바운드 80/443 + **아웃바운드 ephemeral** 확인.

---

## 시나리오

| 시나리오 | 접근 |
|----------|------|
| RDP/SSH 안 됨(Bastion) | **NACL**이 서브넷 아웃바운드 또는 인바운드 차단, **라우팅 테이블**에 IGW/대상 경로 없음. |
| DB 연결 실패(Error establishing connection) | **DB SG**에 웹 서버 소스(또는 SG) **인바운드** 3306 허용, 앱 **포트**가 RDS 포트와 일치하는지 확인. |
| 퍼블릭 웹 서버에 HTTP 접속 불가 | SG 인바운드 80/443, NACL **아웃바운드**에 **ephemeral 포트** 허용. |
| Private 서브넷에서 인터넷( curl ) 실패 | 아웃바운드 **80** 허용(SG). NACL은 0.0.0.0/0 아웃바운드 deny면 차단. |
| NAT 마이그레이션 후 아웃바운드 실패 | NAT 게이트웨이 **가용 상태**, 지원 **프로토콜**(일부 프로토콜 미지원). |
| ALB 타깃 헬스체크 실패 | 리스너 **프로토콜·포트**와 앱이 받는 **프로토콜·포트** 일치. 타깃 그룹 헬스체크 **경로·포트** 확인. |

---

## 기출빈출

| 시나리오 | 접근 |
|----------|------|
| RDP/SSH(Bastion) 안 됨 | **NACL** 인/아웃 차단, **라우팅 테이블**에 IGW·대상 경로 확인. |
| DB 연결 실패(Error establishing connection) | **DB SG** 인바운드에 웹 서버(또는 SG) 3306 허용. 앱 **포트**가 RDS 포트와 일치하는지 확인. |
| NACL이 응답 트래픽 차단 | NACL **아웃바운드**에 **ephemeral 포트**(1024–65535) 허용. |
| Private 서브넷 아웃바운드(인터넷) 실패 | SG **아웃바운드** 80/443. NACL 아웃바운드·라우팅(NAT) 확인. |
| 새 서브넷 추가 후 DB 연결 끊김 | DB 서브넷 NACL에 **새 웹 서브넷**에 대한 인바운드 3306·아웃바운드 ephemeral 추가. |
| ALB 타깃 unhealthy | 앱이 리스너 **프로토콜·포트**에서 동작하는지, 헬스체크 **경로·포트** 일치 확인. |

---

## 요약

| 항목 | 내용 |
|------|------|
| NACL | Stateless. 인바운드+아웃바운드 모두 필요. 아웃바운드 ephemeral 포트 필수. |
| SG | Stateful. 인바운드 허용 시 응답 아웃바운드 자동. 아웃바운드 80/443 등 필요 시 명시. |
| 순서 | Flow Logs로 거절 확인 → NACL(서브넷) → SG(인스턴스) 순으로 점검. |
| 기출 포인트 | NACL·라우팅, DB SG·포트, NACL 아웃바운드 ephemeral, Private 아웃바운드, NAT, ALB 헬스체크. |
