# S3 개요

**객체 스토리지** 서비스로, 일관성·스토리지 클래스·버전·수명 주기·암호화·복제 등으로 비용·내구성·보안을 설정합니다.

---

## 1. Consistency 모델

**PUT·덮어쓰기·삭제 직후** 같은 키로 GET하면 **최신 결과가 보장**되는 읽기 일관성 동작입니다.

- **Strong Read-After-Write**(현재 기본): PUT/OVERWRITE/DELETE 직후 같은 키로 GET 시 최신 결과 보장
- **멀티파트 업로드**: 업로드 완료 후 목록(List) 일관성은 약간의 지연 가능
- **버전링** 사용 시: 이전 버전 ID로 읽으면 해당 버전이 보장됨

| 항목 | 설명 |
|------|------|
| 읽기 일관성 | PUT·덮어쓰기·삭제 직후 같은 키 GET 시 최신 결과 보장 |
| 모델 | Strong Read-After-Write(현재 기본) |

---

## 2. 스토리지 클래스

**접근 빈도·내구성·비용**에 따라 객체를 다른 스토리지 클래스에 두는 기능입니다. 자주 쓰는 데이터는 Standard, 덜 쓰는 데이터는 IA·Glacier 등으로 두어 비용을 줄입니다.

- **S3 Standard**: 자주 접근, 높은 가용성·내구성, 기본 비용
- **S3 Intelligent-Tiering**: 접근 패턴 모름 → 자동으로 자주/덜 쓰는 티어 이동
- **S3 Standard-IA**: 덜 자주 접근, 검색 시 약간의 비용
- **S3 One Zone-IA**: 한 AZ에만 저장, IA보다 저렴
- **Glacier / Glacier Deep Archive**: 아카이브·장기 보관, 검색 지연·복구 시간 있음

**선택 흐름**: 자주 읽기 → Standard / 패턴 불명 → Intelligent-Tiering / 가끔 읽기 → Standard-IA / 아카이브 → Glacier 계열

| 클래스 | 용도 | 비용·특성 |
|--------|------|-----------|
| Standard | 자주 접근 | 기본 가격 |
| Intelligent-Tiering | 접근 패턴 변동 | 자동 티어 이동 |
| Standard-IA | 가끔 접근 | 검색 비용 있음 |
| Glacier 계열 | 아카이브 | 복구 시간·저장 비용 낮음 |

---

## 3. Versioning

같은 **키에 여러 버전**을 두고, 덮어쓰기·삭제 후에도 **이전 버전 ID로 조회·복구**할 수 있게 하는 기능입니다.

- 버전링 활성화 시 객체 **덮어쓰기·삭제**해도 이전 버전 ID로 조회·복구 가능
- 삭제 시 **삭제 마커**만 붙고 버전은 유지(복구 가능)
- 용도: 실수 덮어쓰기·삭제 복구, 규정 준수(이력 보존)

| 항목 | 설명 |
|------|------|
| 동작 | 버전링 ON 시 덮어쓰기·삭제해도 이전 버전 ID로 조회·복구 가능 |
| 삭제 | 객체 삭제 시 삭제 마커만 붙고 버전은 유지(복구 가능) |

---

## 4. Lifecycle

**객체 나이·접두사**에 따라 스토리지 클래스 전환이나 삭제를 **자동 적용**하는 버킷 규칙입니다. 비용 절감·데이터 정리에 사용합니다.

- **Transition**: 일정 일수 후 다른 스토리지 클래스로 이동 (예: Standard → IA → Glacier)
- **Expiration**: 일정 일수 후 객체 삭제 또는 이전 버전 삭제(로그·임시 데이터 정리)

| 구분 | 설명 |
|------|------|
| Transition | N일 후 스토리지 클래스 자동 전환(비용 절감) |
| Expiration | N일 후 객체·이전 버전 삭제(정리) |
| 적용 단위 | 버킷 또는 접두사(prefix) 단위 |

---

## 5. SSE (서버 측 암호화)

S3 **서버 측 암호화(SSE)** 의 두 방식입니다. 키를 S3가 관리할지(SSE-S3), KMS로 제어·감사할지(SSE-KMS) 선택합니다.

- **SSE-S3**: S3가 키 관리, 별도 설정 최소, 버킷 기본 암호화로 설정 가능
- **SSE-KMS**: KMS(CMK)로 암호화, 감사(CloudTrail)·키 정책·권한 제어 가능, SSE-S3보다 비용·제한 있음

| 구분 | SSE-S3 | SSE-KMS |
|------|--------|---------|
| 키 관리 | S3 | KMS |
| 감사·제어 | 제한적 | 상세(CloudTrail 등) |

---

## 6. CRR / SRR (복제)

**다른 리전(CRR)** vs **같은 리전(SRR)** 으로 객체를 비동기 복제하는 기능입니다. **버전링**이 필요하며, DR·로그 집계 등에 사용합니다.

- **CRR**: 다른 리전 버킷으로 비동기 복제 — DR, 리전별 읽기
- **SRR**: 같은 리전 내 다른 버킷으로 복제 — 로그 집계, 프로덕션/테스트 분리
- 공통: 버전링 필요, 소스/대상 버킷 정책 설정, 복제 후 소유권 변경 옵션 등

| 구분 | CRR | SRR |
|------|-----|-----|
| 대상 | 다른 리전 | 같은 리전 |
| 용도 | DR, 리전 분산 | 로그·환경 분리 등 |
