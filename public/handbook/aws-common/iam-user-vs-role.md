# IAM User vs Role

**User**는 한 **사람·프로세스**에 장기 자격을 주고, **Role**은 **임시로 맡아 쓰는** 권한 단위입니다.  
사람/CLI는 User 또는 Role(AssumeRole), AWS 서비스(EC2·Lambda 등)가 다른 서비스에 접근할 때는 **Role**을 쓰는 것이 권장됩니다.

---

## 1. IAM User

- **장기 자격 증명**(Access Key, 비밀번호) 보유
- 사람 로그인·CLI·CI 등에 사용
- 한 사용자에게 여러 정책 직접 부여

---

## 2. IAM Role

- **자격 증명을 갖지 않음** → “이 역할을 맡을 수 있는 주체”가 AssumeRole로 **임시 자격** 발급
- **EC2 인스턴스 역할**, Lambda 실행 역할 등: 서비스가 “이 역할을 맡고” 다른 AWS API 호출
- 크로스 계정·연합 사용자도 Role로 접근

---

## 3. 비교 및 요약

| 구분 | User | Role |
|------|------|------|
| 자격 | 장기(키·비밀번호) | 없음, Assume 시 임시 발급 |
| 용도 | 사람·CLI·고정 프로세스 | 서비스 간·임시 위임·크로스 계정 |
| 권장 | 사람 접근 | 서비스가 다른 AWS 리소스 접근 시 Role |
