# 복제 (Replication)

**원본 데이터를 다른 위치·노드에 지속적으로 동기화**하는 방식입니다. 백업과 달리 실시간에 가깝게 복사본을 유지해 RPO를 줄이고, 장애 시 전환 시간(RTO)을 단축할 수 있습니다.

## 정의

- **복제**: 주(Primary) 데이터의 변경을 대기(Replica) 쪽에 반영해 **동일한 데이터**를 여러 곳에 유지
- **동기 복제**: 쓰기 시 복제 완료까지 확인 후 응답 → 일관성 높음, 지연·가용성 영향
- **비동기 복제**: 쓰기 후 복제는 뒤따라 수행 → 지연 적음, 순간적 데이터 차이 가능

## RPO·RTO와의 관계

- **RPO**: 복제 지연(리플리케이션 랙)만큼 데이터 손실 가능. 동기 복제는 RPO 0에 가깝게 설계 가능
- **RTO**: 이미 복제본이 있으므로 **전환(Failover)** 만 하면 되어, 백업에서 복원하는 것보다 RTO를 짧게 가질 수 있음
