# Routing Table 구조

패킷이 **어느 다음 홉으로 갈지** 결정하는 테이블입니다.

## 역할

- 각 목적지(또는 목적지 범위)에 대해 **다음 홉(Next Hop)** 또는 출구(인터페이스)를 정의
- 라우터·L3 장비가 패킷의 목적지 IP를 보고 테이블을 조회해 전달

## 구조(개념)

| 목적지(CIDR 등) | 다음 홉 / 출구 |
|-----------------|----------------|
| 10.0.0.0/24     | 로컬 / 직접 연결 |
| 0.0.0.0/0       | 기본 게이트웨이   |

- **기본 라우트(0.0.0.0/0)**: 다른 항목에 맞지 않을 때 사용하는 출구

```mermaid
flowchart LR
  P[패킷] --> L[라우팅 테이블 조회]
  L --> N[다음 홉 결정]
  N --> F[전달]
```

## 요약

- 라우팅 테이블 = 목적지 → 다음 홉 매핑
- 서브넷·VPC에서도 동일 개념으로 트래픽 경로가 결정됨
