# Lambda Concurrency

**동시에 실행되는 인스턴스 수**를 제어하는 개념입니다.

## 동시 실행

- 요청마다 인스턴스가 할당되며, **동시 실행 수**가 계정·함수별 예약 동시성으로 제한됨.
- 예약 동시성: 해당 함수가 쓸 수 있는 동시 실행 상한.

## 예약 동시성

- 함수별로 **예약**하면 다른 함수에 영향 줄이거나, 스로틀링으로 다운스트림 보호 가능.
- 예약 0 = 해당 함수 호출 불가(비활성화용).

## 요약

- Concurrency = 동시 실행 수. 예약 동시성으로 함수별 상한·격리 가능.
